# AGENTS.md（项目级工作约定）

本文件用于指导 Codex CLI 在本仓库中的协作方式与交付规范。

## 语言与沟通

- 只使用中文进行沟通、解释与总结。
- 新增/修改的注释与文档默认使用中文。

## 交付与提交（必须遵守）

- 每次任务完成后都需要执行一次 `git commit`（除非用户明确要求“不提交”或要求把多个任务合并为一次提交）。
- 每次提交必须附带“一句解释”（说明本次变更解决了什么/为什么这么改）。
  - 推荐做法：提交信息使用“标题 + 空行 + 一句解释”的格式（把解释放在 commit body 第一行）。
  - 例：
    - 标题：`fix(server): 修复启动参数解析`
    - 解释：`原因：避免在缺少环境变量时启动失败，并给出明确错误提示。`

## 提交前检查（建议）

- 提交前先运行与改动相关的最小验证（如项目已有测试/构建脚本则优先使用）。
- 提交前确保 `git status` 干净（只包含与本任务相关的改动）。

## 提交范围（建议）

- 单次提交尽量聚焦一个问题/一个目标；避免把无关格式化或重构与功能修复混在一起。
